Documentation for Project 2

Process

  • Proposal

    Used Google Slides to create a mockup and used PokeAPI to see how it works and is structured

  • Prototype

    Compleated the catching section as that is the core functionality. Pokedex was functional, with some of the filters created. Favorites was not done as catching and the Pokedex was more important for the prototype. Simple CSS was applied to give some idea of the final site.

  • Final

    More filters added for the pokedex alongside an image and name of the hovered over pokemon. Star added that when clicked will favorite the pokemon displayed above the star (pokemon must be caught). Favorites section shows every pokemon favorited. CSS applied to make the site look like the proposal mockup. Local storage saves the previously searched filters, caught pokemon, caught shiny pokemon, and favorited pokemon.

    One error was found durring the in-class critique, where the Crome would not have enough resources while getting the images for each pokemon. I tried multiple ways to solve this, but none would work. PokeAPI has no good way to get images to every form, since the url's for these images are not standarized. Image IDs were sometimes different to the form IDs, and in some cases the url had the form name after the ID of the default form instead. I also tried to get the images synchronously, but then the site wouldn't load on my slower internet at home (but would work at school). The error is still there, some functionality still works with the error. The school computers worked when using Firefox.
